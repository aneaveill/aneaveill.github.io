Computer Science 1

Course Description

Computer Science 1 is designed to give students foundational skills applicable to all programming languages, but will focus specifically on object oriented programming. Students will apply programming concepts to the development of projects using Java and/or C#. Topics covered will include datatypes, variables, boolean logic, conditional statements, loops, String objects, and functions/methods.

Variables, Datatypes, and I/O

Unit 1 Overview

In this unit you will start to become acquainted with foundational concepts in Java.


Objectives

  • Students will be able to output text to the console
  • Students will be able to store values into variables using appropriate datatypes
  • Students will be able to create and use basic methods/functions (no parameters/returns)
  • Students will be able to prompt for user input and store it for future use
  • Students will be able to write professional-looking, organized code with comments and adequate spacing

Power Standards

  • Output (Print)
  • Variables/Datatypes
  • Methods/Functions (without parameters/returns)
  • Input (Scanner)
  • Coding Etiquette / Best Practices
    • Note: This will just be the general knowledge portion of the etiquette, as the course progresses ALL code is subject to the etiquette standard and points can be reduced for breaches of general coding etiquette.

Unit 1 Slides

Assignments/Practice

  • PracticeIT (Binary Intro)
    • Allows students become acquainted with the PracticeIT platform and get a little practice with binary values
    • Building Java Programs, 5th Edition, Chapter 1, Self-Checks 1-3
  • PracticeIT (Output Practice)
    • A series of problems allowing for the practice using System.out.print() and System.out.println()
    • Building Java Programs, 5th Edition, Chapter 1, Self-Checks 7-10
  • PracticeIT (Binary Intro)
    • Allows students become acquainted with the PracticeIT platform and get a little practice with binary values
    • Building Java Programs, 5th Edition, Chapter 1, Self-Checks 1-3
  • Practice Lab - "Introduce Yourself"
    • Your very first program! This will allow you to expand on the "Hello World" program and give it your own personal touch. Additionally this will give you practice using an Online IDE
    • Your task will be to write a program that prints output introducing yourself with at least 5 lines of output.
  • Exact Instruction Activity
    • Gives you some practice being precise with your language of, details of, and order of your instructions. This is intended to give you some insight as to how the compiler will treat your code.
    • Your task will be to partner up with someone in the room. After watching a video of how instructions can be... misinterpreted, you each write your own set of instructions to make a peanut butter and jelly sandwich. Then, your partner must pick apart those instructions for any amount of vagueness or something that can be misunderstood.
  • PracticeIT (Basic Methods/Functions)
    • In this practice, you will get to write methods/functions and call methods/functions
    • Remember, the main() method is the only one the program runs to start, and the order things are called within main() or any function within the main() matters.
    • Building Java Programs, 5th Edition, Chapter 1, Self-Checks 21-28
  • PracticeIT (Code Cleanliness)
    • This practice is simply showing you how messy code can lead to problems. You will be asked to make the code more presentable/readable.
    • Building Java Programs, 5th Edition, Chapter 1, Self-Checks 30-32
  • PracticeIT (Math Operators)
    • This is practice using the math operators in Java. The focus here is order of operations (P-E-MMD-AS, yes that extra M is on purpose for modulus % ).
    • Remember, Math operations only carry the decimal through to the result if there was a decimal on either operand to begin with. Ex. 3/2 is 1, but 3.0/2, 3/2.0, and 3.0/2.0 all result in 1.5
    • Also remember that a String (something in quotes) + anything results in a String combining the two. "hi" + 5 is "hi5", 2+"4" is 24, etc. Make sure you are working one operation at a time, with order of operations, left to right
    • Building Java Programs, 5th Edition, Chapter 2, Self-Checks 1-5
      • Note: on problems 3-5 you only need to evaluate 7 math expressions from each page, not the whole set.
  • PracticeIT (Variables and Datatypes)
    • In this one, the practice focuses on declaring and using variables with appropriate datatypes
    • Building Java Programs, 5th Edition, Chapter 2, Self-Checks 6-10
  • PracticeIT (Math with Variables)
    • Now we are practicing math with variable assignment!
    • Building Java Programs, 5th Edition, Chapter 2, Self-Checks 13-17
  • Scanner Exit Ticket
    • The focus of this exit ticket is to make sure you really understand how to import, declare and use a Scanner to get input
    • The task is to write a small program that will ask the user for their favorite food. You then store that input into a variable and show that it was correctly scanned by using that variable in a sentence agreeing. Ex. "I think _______ is the best food ever" but the blank should instead be the value of their scanned input.
  • Unit 1 Study Guide
    • Students will get a list of focus topics for the exam so they can compile notes, practice, and examples to assist them on the exam.

Booleans, Conditionals, and Strings

Unit 2 Overview

In this unit you will start learn how to guide your program to make decisions based on true/false statements. You will also learn how to further manipulate text in Java.


Objectives

  • Students will be able to create and evaluate true/false statements with various operators
  • Students will be able to create and evaluate code that uses conditional statements (if, if->else, if->else-if->else
  • Students will be able to use methods within the String class to read and manipulate text Strings

Power Standards

  • Booleans
  • Conditionals
  • String Methods

Unit 2 Slides

Assignments/Practice

  • Boolean Exit Ticket
    • A short series of questions where you are given values for 3 variables (x, y, and z), and using those evaluate a series of expressions that yield true or false
  • Trux Falsy Practice
    • Trux-Falsy is a visual practice for boolean expressions
    • Students need to make their way through all challenges up through Generation 2, Litter 4
    • No need for 3 star solutions, that just means it was the most efficient/least calculation solution
    • Student can go beyond those problems if they want more practice
  • Boolean Bingo
    • A short social activity where students need to fill out a bingo board by interpreting boolean statements and finding classmates for which those statements are true
  • Conditional Exit Ticket
    • A brief check to make sure students are able to write an if-else conditional statement.
  • PracticeIT (Conditionals)
    • In this practice, studets will get to practice writing conditionals to meet specific criteria.
    • Building Java Programs, 5th Edition, Chapter 4, Self Checks 7-11
  • String Exit Ticket
    • A brief check where students are asked to manipulate Strings using String methods to make sure they understand how to properly implement these methods.
  • Unit 2 Study Guide
    • Students will get a list of focus topics for the exam so they can compile notes, practice, and examples to assist them on the exam.

Assessments

  • Boolean Quiz
    • A short quiz over boolean statements. Students will be expected to both construct and interpret boolean statements
    • Assesses the following standards:
      • Booleans
  • Favorite Things Project
    • Students will create a program that asks the user what their favorite ______ is (filling in the black with some category like food, color, movie, game, etc). If the user's input matches the programmer's favorite, the program says something in response to acknowledge that. This output should be different if it doesn't match.
    • Students can use the Scanner Exit Ticket as a baseline, but will need to expand it to decide the output using a conditional, and will need String mathods to compare Strings correctly within the condition.
    • Students can further customize it and show their skills by changing to match multiple different inputs with unique outputs. They can also use different String methods to accept variations on input (ex. Favorite color of "blue" or "Royal Blue" or "BLue" would all be accepted as "Blue").
    • Assesses the following standards:
      • Conditionals
      • String Methods
  • Unit 2 Exam
    • Assesses all standards from unit 2

Loops

Unit 3 Overview

In this unit you will learn how to use loops to repeat and iterate sections of code.


Objectives

  • Students will be able to write code that uses loops to repeat code
  • Students will be able to write and trace `for` loops
  • Students will be able to write and trace `while` loops

Power Standards

  • Loops

Unit 3 Slides

Assignments/Practice

  • PracticeIT (For Loop Creation)
    • This is a set of practice problems where students will be asked to write `for` loops
    • Building Java Programs, 5th Edition, Chapter 2, Self-Checks 21-23
  • PracticeIT (For Loop Tracing)
    • This is a set of practice problems where students will be asked to trace `for` loops
    • Building Java Programs, 5th Edition, Chapter 2, Self-Checks 24-28
  • While Loop Tracing Exit Ticket
    • Students will be given a brief looping task that must be written using a `while` loop

Assessments

  • Looping Project
    • This project will require students to use loops to address a task of their choosing.
    • Students can choose their own level of difficulty by picking one of two tasks
        • Times Table Generator
        • Students will prompt the user for a number and how many multiples of that value they want to see. The program will then display those values.
        • Example:
          Enter a number: 5
          How many multiples do you wish to see? 10
          Result: 5 10 15 20 25 30 35 40 45 50
    • Assesses the following standards:
      • Loops
  • Unit 3 Quiz
    • Assesses all standards from unit 3
